# 一、整体架构（分层设计）

```Plain Text
┌─────────────────────────────────────────────────────┐
│                 用户接口层 (Client API)              │  ── 对外提供 Put/Get/Delete/Scan
├─────────────────────────────────────────────────────┤
│                 事务/原子操作层                      │  ── 原子写、批量操作、快照
├─────────────────────────────────────────────────────┤
│                 写入流水线 (Write Path)             │
│  WAL → MemTable → Immutable MemTable → Flush → L0   │
├─────────────────────────────────────────────────────┤
│                 读取流水线 (Read Path)              │
│  MemTable → Immutable → Level 0~N → BlockCache      │
├─────────────────────────────────────────────────────┤
│                 Compaction 合并调度器                │  ── 核心：控制写放大/读放大
├─────────────────────────────────────────────────────┤
│              磁盘文件管理层 (SSTable/Manifest)      │  ── 文件格式、元数据、版本管理
├─────────────────────────────────────────────────────┤
│              通用工具 & 高性能组件                   │
│  跳表 / 布隆过滤器 / 编码 / 校验 / 内存池 / 锁      │
└─────────────────────────────────────────────────────┘
```

**架构目标**

- 极高写入吞吐（顺序写）

- 低读取延迟（缓存 \+ 索引 \+ 布隆）

- 稳定低抖动（Compaction 调度优化）

- 崩溃安全（WAL \+ 原子刷盘）

- 可观测、可配置、可测试

# 二、必须实现的**全部核心模块**（工业级完整清单）

## 1\. 基础数据结构（核心性能底座）

### 1\.1 跳表 SkipList

- 用途：**MemTable 底层结构**

- 必须实现：

    - 有序插入

    - 范围查找（Scan）

    - 并发安全（读写锁或无锁）

    - 迭代器

- 工业级要求：

    - 层高自适应

    - 内存紧凑

    - 支持快照迭代

### 1\.2 布隆过滤器 BloomFilter

- 用途：**过滤不存在的 Key，大幅减少磁盘 I/O**

- 必须实现：

    - 位图 bit array

    - 多个哈希函数

    - SSTable 内置

- 优化：

    - 高性能哈希（xxhash/murmurhash）

    - 可配置误判率

### 1\.3 LRU 缓存 BlockCache

- 用途：缓存 SSTable 数据块，加速读

- 必须实现：

    - LRU 淘汰

    - 并发安全

    - 分片锁（避免全局锁竞争）

    - 内存上限控制

### 1\.4 编码层（Key/Value 序列化）

- 变长编码（varint）

- 键值对紧凑存储

- 前缀压缩（减少空间放大）

---

## 2\. 写入路径核心模块

### 2\.1 WAL（Write Ahead Log）

**崩溃恢复唯一保证**

- 顺序追加写

- 块式写入

- 校验和（CRC32）

- 日志滚动、清理

- 崩溃恢复重放

### 2\.2 MemTable

- 跳表封装

- 容量控制

- 写入并发安全

- 支持 Get / Iterate

- 墓碑标记（Delete 实现）

### 2\.3 Immutable MemTable

- 只读、不可修改

- 异步 Flush 到磁盘

- 防止写入阻塞

### 2\.4 Flush 机制

- 后台线程池异步执行

- 生成 SSTable 并写入 L0

- 原子切换（保证崩溃安全）

---

## 3\. 磁盘存储核心

### 3\.1 SSTable（Sorted String Table）

**LSM 磁盘文件标准格式（工业级必须完整实现）**
一个 SSTable 包含：

1. **Data Block**（真实键值数据）

2. **Filter Block**（布隆过滤器）

3. **Index Block**（key → block 偏移）

4. **Footer**（固定大小，定位元数据）

必须实现：

- 块压缩（zstd/lz4）

- 前缀压缩

- 二分查找

- 数据校验

- 懒加载（mmap 或按需读取）

### 3\.2 Manifest（元数据管理）

- 记录所有层级的 SSTable 文件

- 记录版本变更

- 宕机后重建目录树

### 3\.3 Version \+ VersionSet（版本管理）

**工业级必备**

- 多版本并发读（MVCC 思想）

- Compaction 期间不阻塞读写

- 原子提交层级变化

---

## 4\. Compaction（LSM 最核心、最难模块）

### 4\.1 分层结构 Leveled Compaction（默认工业标准）

```Plain Text
L0: 无序、文件重叠
L1~LN: 内部无重叠、有序、大小逐层翻倍
```

### 4\.2 必须实现功能

- 多路归并排序（k\-way merge）

- 墓碑清理（删除真正生效）

- 旧版本覆盖数据清理

- 层级选择策略

- 写暂停控制（防止 Stall）

### 4\.3 Compaction 调度器

- 动态选择合并文件

- IO 限速

- 线程池隔离

- 避免写抖动

---

## 5\. 读取路径

### 5\.1 Get 流程（必须完整）

MemTable → Immutable → L0 → L1 → … → LN

- 布隆过滤器快速过滤

- 索引二分查找

- 缓存命中直接返回

### 5\.2 Iterator / Scan（范围查询）

- 全存储统一迭代器

- 多层归并迭代

- 前缀扫描

- 快照一致性读

### 5\.3 快照 Snapshot

- 读取一致性保证

- 不阻塞 Compaction

- 多版本读取

# 四、三大放大系数（LSM 衡量质量核心）

你必须实现统计：

1. **写放大 Write Amplification**
写入 1KB 业务数据，实际磁盘写多少

2. **读放大 Read Amplification**
一次 Get 访问多少个 SSTable

3. **空间放大 Space Amplification**
有效数据 / 实际占用空间
